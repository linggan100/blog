<a>=链接？你错了
发布于：2019-11-27 标签：/ javascript / 访问：

通常情况，我们知道一个写了<a>标签的地方应该是个链接，点击会产生跳转的动作，当然，我们也可以不使用<a>，用脚本完成同样的功能。但<a>的用途仅限于此吗？

当然不是，其实你可能已经体验过它们，只是没有深究过它怎样具备了那样的能力，本文我们就来窥探一番。

URL

先看最常见的，示例如下：

<a id="test" href="http://ideazhao:passwd@example.com:8081/index.html?order=1#detail">test</a>

这样的url你可能很熟悉，我们通过Location对象的属性提取细节。

var a = document.getElementById('test');
a.hash // "#detail"
a.hostname // "example.com
a.password // "passwd"
a.pathname // "/index.html"
a.port // "8081"
a.search // "?order=1"
a.username // "ideazhao"

这些能力，你可能不知道，也或许不知道，但没关系，现在你知道了。

accessKey（快捷键）

我们每天都在使用快捷键，但你想过可以用js代码去设置我们想要的快捷键来完成跳转么？还是上面的示例。

var a = document.getElementById('test');
a.accessKey = 'g';

如果你忍不住试了一下会发现，怎么摁“g”都没有反应，是的，快捷键跟所使用的浏览器或者系统是相关的，像我们使用的其他快捷键一样，通常需要配合ctrl或者alt等功能键一起才会生效，mac中的chrome浏览器就是“Control + Alt + key”，具体规则可参考此链接 ——accesskey

你可能有个疑问，既然是属性，可否直接在标签上设置呢？答案是可以，像这样

<a href="https://www.baidu.com/" accessKey='g'>跳转百度</a>

也是生效的。

download（下载）

download属性表示当前链接不是用来浏览，而是用来下载的。它的值是一个字符串，即用户下载得到的文件名。
