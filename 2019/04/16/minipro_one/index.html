小程序由浅入深（一）
发布于：2019-04-16 00:00:00 标签：/ 学习、小程序 / 访问：

小程序登场已两年有余，它出现之后曾经有过非常多文章解读和赞美，随之而来的，网页开发领域又多出一项需求——小程序开发。那么小程序究竟是什么样子，能做什么，以及怎么做呢？这篇文章开始，我们来逐渐深入地了解。

是什么

小程序，从名字能反应出第一个特点，体量小，相比那些动辄几百M甚至几G的应用来说，它会小很多。

当然还有另外一个特点，是不需要下载安装，它是依托于某个平台的，比如现在存在的“微信、百度、支付宝”等，那些原本是独立应用的，变成了这些应用里面的网页一般，用的时候打开，不用的时候关掉。

虽然这么说，它跟普通的网页开发又有着区别，比如：运行环境、运行机制等，这就导致有些东西的使用方法发生了改变，甚至不可用，具体后面会看到。

遛一遛

一些程序上的东西就不多说了，来直接看看一个小程序有哪些内容以及分别代表什么。

一、json配置

app.json

能看出来，这应该是整个app的全局配置，哪些属于全局呢？

比如，项目有哪些文件,界面框架的UI,底部tab等。看代码

{
"pages":[
    "pages/index/index",
    "pages/contact/contact",
    "pages/logs/logs"
],
"window":{
    "backgroundTextStyle":"light",
    "navigationBarBackgroundColor": "#000",
    "navigationBarTitleText": "lingganzhijian",
    "navigationBarTextStyle":"white"
},
"tabBar": {
    "list": [
    {
        "pagePath": "pages/index/index",
        "text": "首页"
    },
    {
        "pagePath": "pages/contact/contact",
        "text": "contact"
    }
    ]
}
}

这样，项目的整体框架算是“撘”好了，有哪些页面文件，窗体的表现，底部导航的内容配置，有一个小惊喜就是，小程序的页面可以像app那样去定制自己的title区域了。

但这只是一个很简单的开始，想把页面丰富起来，需要我们往里填充内容。

小程序也是程序，类似网页，那就也有三剑客供使用，只是名称和写法有不同，这里以微信小程序为例，它们是：
js、wxml、wxss，对应的就是js、html和css。
