【JS启示记】对象的隐式属性
发布于：2020-03-10 标签：/ 学习、javascript / 访问：

什么是”隐式属性“？它有，你却看不见。

不仅看不见，轻易也不会去使用，那么它们用处大么，当然！

下面就来简要介绍（非全部）：

纯粹的Object

  • constructor

用于保存创建当前对象的函数，即我们通常说的”构造函数“。

  • isPrototypeOf()

用于检查传入的对象是不是另一个对象的原型

仍拿上例来说

Person.prototype.isPrototypeOf(liming); // true
  • propertyIsEnumerable(propertyName)

检查给定的属性是否可以使用 for-in 语句枚举。

  • hasOwnProperty(propertyName)

有时候会使用 for-in 语句遍历检查一个对象是否具有某个属性，但这个方法不仅能查找到当前对象的属性，还能查到原型链上的对象是否有，如果要把那些排除掉，就可以使用hasOwnProperty()方法。

  • toString()

返回对象的字符串表示。

这个方法我们很常见，几乎是转换字符串最常用的方法了，但你会发现从来没有定义过。

其一、自带无需定义，其二、重新定义或者更改内置方法容易造成某些未知的问题。

但它在多数对象中均被覆盖，就得到了不同的值了，也是人们预期的结果。

  • valueOf()

返回对象的字符串、数值或者布尔值，也就是说，对象本身的”值“或者称之为”内容“是什么，就返回什么，我们依然拿上面示例测试一下
